## 🚀 Projetos em Destaque

### 1. MVP de IA para Modernização e Governança em Engenharia Automotiva

* **O Problema:** Uma grande operação de engenharia automotiva coletava dados de testes e laudos de forma manual. Informações sensíveis e proprietárias de novos produtos transitavam por canais de comunicação não oficiais, gerando risco crítico de vazamento de dados, baixa rastreabilidade e retrabalho operacional.

* **A Solução:** Idealizei e desenvolvi do zero um MVP em formato SPA/PWA integrado ao Firebase. Criei o módulo "Smart Laudo", com integração de LLMs via API, engenharia de prompts estruturada e higienização de inputs com abordagem Security by Design.

* **A Arquitetura Técnica:**
  * **Frontend:** Aplicação SPA/PWA com JavaScript, HTML e CSS.
  * **Backend/Cloud:** Firebase para autenticação, persistência e estruturação inicial dos dados.
  * **Automação:** Criação de pipelines inteligentes com n8n para automação de fluxos e ingestão de dados.
  * **IA:** Integração de APIs de LLMs para análise automatizada de laudos.
  * **Segurança:** Higienização de inputs, proteção de credenciais, redução de exposição de dados sensíveis e alinhamento com princípios de LGPD.

* **O Resultado:** Digitalização de coletas manuais, redução do tráfego de dados sensíveis por canais inseguros e centralização das issues técnicas em uma plataforma única e auditável, demonstrando o valor da IA aplicada à operação.

* **Stack:** `JavaScript` `Firebase` `HTML` `CSS` `n8n` `LLM APIs` `Prompt Engineering` `Security by Design`

* **Documentação:** [PDF do Documento Tecnico TPG Insight AI](https://drive.google.com/file/d/1w8E1HnuSv6AcBXe4cVhYSE5v5cmoe52B/view)

### 2. Sistema Seguro de Gerenciamento de Tarefas com DevSecOps

* **Período:** fev/2025 – abr/2025

* **O Problema:** Aplicações web simples muitas vezes são desenvolvidas sem práticas adequadas de segurança, rastreabilidade, automação de testes, análise de vulnerabilidades e monitoramento pós-implantação.

* **A Solução:** Desenvolvi do zero um sistema web seguro de gerenciamento de tarefas pessoais, aplicando práticas completas de SDLC, DevOps e DevSecOps. A aplicação incluiu autenticação obrigatória, CRUD de tarefas, modelagem de fluxos de uso, análise de ameaças, uso de variáveis de ambiente, criptografia e geração de logs operacionais.

* **A Arquitetura Técnica:**
  * **Backend:** Python, Flask, SQLite, SQLAlchemy e WTForms.
  * **Containerização:** Dockerfile e docker-compose para execução padronizada em containers.
  * **CI/CD:** Pipeline completo no GitLab com etapas de build, testes automatizados com pytest, análise estática com Bandit e análise de dependências com OWASP Dependency-Check.
  * **Segurança:** Testes DAST com OWASP ZAP em container, identificação e correção de vulnerabilidades como ausência de CSP, riscos de XSS e headers inseguros.
  * **Deploy:** Configuração de ambientes de staging e review apps com URLs temporárias para merge requests.
  * **Monitoramento:** Fail2Ban para análise de logs e bloqueio de tentativas suspeitas, além de Prometheus e Grafana para observabilidade.

* **O Resultado:** Criação de uma aplicação web segura, containerizada, testada, monitorada e integrada a um pipeline DevSecOps completo, consolidando práticas reais de segurança ofensiva, segurança defensiva, automação de deploy e observabilidade.

* **Stack:** `Python` `Flask` `SQLite` `SQLAlchemy` `WTForms` `Docker` `GitLab CI/CD` `pytest` `Bandit` `OWASP ZAP` `OWASP Dependency-Check` `Fail2Ban` `Prometheus` `Grafana`
